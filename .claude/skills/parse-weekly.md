# 解析週報 Markdown

## 任務

將 Google Docs 匯出的單一 `.md` 檔案解析為結構化 JSON，用於匯入資料庫 `articles` 表。

## 輸入

一份已處理過的 `.md` 檔案（base64 圖片已轉換為 bucket URL）。

## 文件結構特徵

這份文件包含 **8 個固定分類（category）**，每個分類下有 **1~N 篇文稿**。

### 固定分類對照表（必須使用 category_id）

| category_id | name | 對應版次 |
|-------------|------|----------|
| 1 | 全球焦點 | 一版 |
| 2 | 證嚴上人開示 | 二版 |
| 3 | 慈濟要聞 | 三版 |
| 4 | 慈善志業要聞 | 四版 |
| 5 | 里仁為美 | 五版 |
| 6 | 大醫行願 | 六版 |
| 7 | 春風化雨 | 七版 |
| 8 | 人文馨香 | 八版 |

**重要：不可建立新分類，每篇文稿必須歸入上述 8 個分類之一。根據文稿內容和版次資訊，選擇最相近的 category_id。**

### 分類辨識規則

分類標題通常包含以下特徵（但格式可能不一致）：
- 含有版次資訊：「一版」「二版」...「八版」或「第一版」「1版」等變體
- 含有分類名稱：如「全球焦點」「證嚴上人開示」等
- 可能帶有狀態標記：如「（完稿）」「（定稿）」等
- 可能是 H1（#）但也可能不是——不要依賴 markdown 格式層級來判斷

### 文稿辨識規則

每篇文稿包含：
- **標題**：通常是分類標題之後的下一個標題級文字
- **內文**：標題之後的段落內容，可能包含：
  - 粗體標記的訊息來源（如 `**臺南訊/**`）
  - 小標題（段落內的子標題）
  - 引用文字（斜體或引用區塊）
- **圖片**：以 markdown 圖片語法嵌入（`![caption](url)`），圖片說明文字通常在圖片下方

### 文稿邊界判斷

以下特徵表示一篇文稿結束、下一篇開始：
- 水平分隔線 `---`
- 新的文稿標題出現（語義判斷，非單純靠格式）
- 新的分類標題出現

## 格式修正

在解析過程中，同時修正以下常見問題：
- 錯誤的標題層級（例如全部用 H1）→ 修正為合理的層級結構
- 多餘的空行或空白
- 不一致的圖片語法
- 分類標題格式統一化

## 輸出格式

### 結構化 JSON

```json
{
  "weekly_id": 117,
  "categories": [
    {
      "category_id": 1,
      "name": "全球焦點",
      "sort_order": 1,
      "articles": [
        {
          "title": "慈濟「預鑄安心屋」完工 楠西山城震後新生",
          "content": "**臺南訊/**二○二五年一月廿一日清晨...\n\n![預鑄屋完成](https://bucket-url/image1.jpg)\n預鑄屋完成。（攝影：王永周）"
        },
        {
          "title": "慈濟泰北寒冬送暖...",
          "content": "..."
        }
      ]
    },
    {
      "category_id": 2,
      "name": "證嚴上人開示",
      "sort_order": 2,
      "articles": [...]
    }
  ]
}
```

### 欄位說明

- `weekly_id`：期數（從檔案名稱或內容提取）
- `categories[].category_id`：固定分類 ID（必須從 1-8 中選擇，對應上方分類對照表）
- `categories[].name`：分類名稱（必須與對照表完全一致）
- `categories[].sort_order`：版次順序（一版=1, 二版=2...）
- `articles[].title`：文稿標題（純文字，去除 markdown 格式符號）
- `articles[].content`：文稿完整內容（保持 markdown 格式，包含圖片引用）

## 注意事項

1. **不要信任格式層級**——編輯者可能將所有文字都設為 H1，用語義理解判斷
2. **分類標題 vs 文稿標題**——分類標題含有版次資訊，文稿標題是具體新聞/文章的名稱
3. **圖片歸屬**——圖片屬於其上方最近的文稿
4. **content 不包含 title**——title 已獨立提取，content 從標題之後開始
5. **保持 content 的 markdown 格式**——不要轉換為 HTML 或純文字
6. **圖片說明文字**——圖片下方的說明文字（caption）應包含在 content 中
